Let and const are block-scoped, whereas var is function-scoped. Let and const variables cannot be changed, while var variables may. Let and const variables are declared using the let and const keywords, respectively, while var variables are declared using the var keyword. Let and var variables can be changed, while const variables cannot be changed.

The syntax of arrow functions and normal functions is the first and most visible distinction between them. Arrow functions not only have a distinct appearance, but they also offer an implicit return shorthand and permit the omission of the parenthesis surrounding a single argument. The arguments array is a unique object that can be used to retrieve all of the parameters supplied to a function in JS. In a manner similar to this, arrow functions also do not have their own binding to an arguments object; instead, they are bound to arguments of the surrounding scope.

Unlike earlier JavaScript versions, it does not require concatenation when inserting or interpolating variables and expressions into strings. Using it, we are able to construct complicated strings with dynamic parts.

The map method is used to convert each item of an array, while the filter method is used to select certain items of an array. The main difference between forEach and filter is that forEach just loop over the array and executes the callback but filter executes the callback and check its return value. If the value is true element remains in the resulting array but if the return value is false the element will be removed for the resulting array. The find() method is used to find all the descendant elements of the selected element. It finds the element in the DOM tree by traversing through the root to leaf. The filter() method is used to filters all the elements and returns the element that matches and the element that do not match are removed. The only difference is the filter() method search through all the elements while find() method search through all the child elements only.
